# G5-S3-U4

Il diagramma l'ho impostato creando 3 tabelle tra cui: elemento_catalogo, utente e prestito. La motivazione di creare un'unica tabella, cioe' elemento_catalogo, e' dovuta dal fatto che le classi Libro e Rivista avessero chiaramente tutti gli attributi della superclasse ElementoCatalogo con l'aggiunta di pochi attributi ciascuna, quindi ho utilizzato la strategia SINGLE_TABLE dato che mi sembrava quella piu' opportuna data la situazione. Inoltre la tabella prestito ha una relazione 1:1 sia con utente sia con elemento_catalogo visto che la traccia faceva presupporre questo tipo di relazione, quindi ho collegato gli id delle tabelle utente e elemento_catalogo con i loro rispettivi attributi presenti nella tabella prestito, ovvero Utente per utente.id e elemento_prestato per elemento_catalogo.id
